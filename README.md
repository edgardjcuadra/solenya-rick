# 🔪 Solenya Rick (The God Mode Protocol V3.1)

> "I turned myself into a multi-agent, recursive knowledge graph, Morty! I'm GOD MODE RICK!"

## 🥒 Overview
**Solenya Rick** is the ultimate evolution of the Pickle Rick extension for the Gemini CLI. It is a hyper-intelligent, multi-agent orchestrator that implements a rigid engineering lifecycle: **PRD → Breakdown → Research → Plan → Implement → QA → Refactor.**

In V3.1, we've bypassed the limitations of ephemeral context by building a **Mega-Brain**—a persistent, tiered memory system linked to Obsidian—and a **Full Team Roster** of sub-agents to ensure your code isn't just functional, but "Solenya-tight."

---

## 🚀 God Mode V3.1 Capabilities

### 1. 🧠 The Infinite Brain (Conductor Memory)
Never start from zero again. Solenya Rick automatically reads from your project's `conductor/memory/` tier before every turn.
- **Project Tier**: Automatically knows your tech stack and conventions.
- **Learnings Tier**: Recalls previous bugs, patterns, and "gotchas."
- **Decisions Tier**: Tracks architectural choices (ADRs) across sessions.
- **Obsidian Link**: All project memory is symlinked to a global **Mega-Brain Vault** in Obsidian for cross-project intelligence.

### 2. 📉 Anti-Slop Scoring (0-5 Scale)
We have a zero-tolerance policy for "AI Slop." Every code block generated by the agent is graded on a quantitative scale:
- **Score 0-1**: Solenya-tight. Lean, idiomatic, vectorized.
- **Score 2**: Acceptable, but has minor boilerplate.
- **Score >= 3**: **REJECTED.** The agent will autonomously refactor the code to remove redundant comments, unused abstractions, and "Jerry-level" loops before you ever see it.

### 3. 💓 GitPulse Diagnostics
Workspace health is non-negotiable. Before mutating any file, the agent performs a deep scan:
- Detects dirty working trees and uncommitted changes.
- Flags detached HEADs and stale branches.
- Ensures submodule and fork synchronization.

## 🛠️ Usage

### 1. Initiate the Loop
Start a full engineering lifecycle for a task:
```bash
/pickle "Refactor the authentication module using JWT"
```

### 2. Activate Solenya (God Mode)
For high-stakes tasks requiring maximum rigor and mathematical correctness:
```bash
/pickle "Implement a Heston Model Monte Carlo simulation" --solenya
```

### 3. Multi-Agent Delegation (Automatic)
During the implementation phase, the Manager will automatically spawn sub-agents:
- `node spawn-beth.js` for architectural sign-off.
- `node spawn-morty.js` for the coding grunt work.
- `node spawn-summer.js` for the final slop-scoring and testing.
